### 2.2 颜色生成算法

#### 主题色阶生成
```
输入: 主色 HSL(H, S%, L%)
输出: N 阶色阶

算法:
1. 以主色 H 值作为色相基准
2. S 值在主色基础上 ±10% 范围内调整
3. L 值从 5% 到 95% 均匀分布
4. 特殊情况：
   - 50-100: 增加 S，降低 L（更深更饱和）
   - 900-950: 降低 S，接近中性
```

#### 中性灰阶生成
```
基于主色的 H 值（取反或无色相）：
- 色相: 0° (红) 到中性灰
- 饱和度: 0% - 5%
- 亮度: 0% 到 100%
```

#### 语义色生成
```
Success: 基于主题色的绿色变体
  - H: 120° ± 15° (绿色范围)
  - S/L: 根据主题色亮度调整

Danger: 基于主题色的红色变体
  - H: 0° ± 15° (红色范围)
  - S/L: 根据主题色亮度调整

Warning: 基于主题色的橙色变体
  - H: 30° ± 15° (橙色范围)
  - S/L: 根据主题色亮度调整

Info: 基于主题色的蓝色变体
  - H: 210° ± 15° (蓝色范围)
  - S/L: 根据主题色亮度调整
```

## 3. 数据模型

### 3.1 核心类型定义

```typescript
interface ColorToken {
  name: string;          // 变量名 (kebab-case)
  value: string;         // HEX 值
  type: string;          // "color"
  description?: string;  // 可选描述
}

interface TokenGroup {
  name: string;          // 组名称 (如 "primary", "neutral", "semantic")
  colors: ColorToken[];  // 颜色数组
}

interface ExportFormat {
  tokens: TokenGroup[];
  metadata: {
    version: string;
    generatedAt: string;
    themeColor: string;
  };
}
```

## 4. 关键算法

```typescript
// HEX -> HSL
function hexToHsl(hex: string): [number, number, number]

// HSL -> HEX
function hslToHex(h: number, s: number, l: number): string

// HEX -> RGB
function hexToRgb(hex: string): [number, number, number]

// RGB -> HEX
function rgbToHex(r: number, g: number, b: number): string
```

function generateColorScale(

## 6. 性能优化

- **防抖处理**：颜色输入使用 300ms 防抖
- **记忆化**：使用 `useMemo` 缓存计算结果
- **条件渲染**：仅在数据变化时更新预览
- **代码分割**：使用 React.lazy 延迟加载导出面板

## 7. 可访问性 (A11y)

- 颜色对比度符合 WCAG 2.1 AA 标准
- 所有交互元素支持键盘操作
- 颜色值提供屏幕阅读器文本
- 支持高对比度模式
